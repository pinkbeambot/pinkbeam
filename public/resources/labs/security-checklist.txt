# Security Checklist for Web Applications

## Authentication

### Password Policy
- [ ] Minimum length enforced (12+ characters)
- [ ] Complexity requirements (mixed case, numbers, symbols)
- [ ] Common password blacklist
- [ ] Password history enforced (no reuse)

### Session Management
- [ ] Secure, httpOnly, sameSite cookies
- [ ] Session expiration configured
- [ ] Session invalidation on logout
- [ ] Concurrent session limits

### Multi-Factor Authentication
- [ ] MFA available for all users
- [ ] TOTP or hardware key support
- [ ] Backup recovery codes
- [ ] Enforced for admin accounts

## Authorization

### Access Control
- [ ] Principle of least privilege
- [ ] Role-based access control (RBAC)
- [ ] Resource-level permissions
- [ ] Regular access audits

### API Security
- [ ] Authentication on all endpoints
- [ ] Authorization checks for each action
- [ ] Rate limiting implemented
- [ ] CORS properly configured

## Data Protection

### Sensitive Data
- [ ] Encryption at rest (AES-256)
- [ ] Encryption in transit (TLS 1.3)
- [ ] Field-level encryption for PII
- [ ] Data masking in logs

### Database Security
- [ ] Parameterized queries (prepared statements)
- [ ] Connection encryption enforced
- [ ] Access restricted by IP
- [ ] Regular backups encrypted

## Input Validation

### Client-Side
- [ ] Form validation feedback
- [ ] File type and size validation
- [ ] Sanitized rich text input

### Server-Side
- [ ] All inputs validated
- [ ] Whitelist validation preferred
- [ ] SQL injection prevention
- [ ] XSS prevention (output encoding)
- [ ] CSRF tokens on state-changing requests
- [ ] File upload restrictions

## Infrastructure

### Network Security
- [ ] Web Application Firewall (WAF)
- [ ] DDoS protection
- [ ] Network segmentation
- [ ] VPN for internal access

### Headers & Configuration
- [ ] Content Security Policy (CSP)
- [ ] X-Frame-Options header
- [ ] X-Content-Type-Options header
- [ ] Strict-Transport-Security (HSTS)
- [ ] Referrer-Policy header

## Monitoring & Response

### Logging
- [ ] Security events logged
- [ ] Failed login attempts tracked
- [ ] Privilege escalation logged
- [ ] Logs tamper-protected

### Incident Response
- [ ] Incident response plan documented
- [ ] Contact information updated
- [ ] Automated alerting configured
- [ ] Regular drill exercises

## Compliance

### Data Privacy
- [ ] GDPR compliance (if applicable)
- [ ] CCPA compliance (if applicable)
- [ ] Privacy policy updated
- [ ] Data retention policy enforced

### Security Testing
- [ ] Regular penetration testing
- [ ] Automated vulnerability scanning
- [ ] Dependency vulnerability monitoring
- [ ] Security code reviews

## Third-Party Components

- [ ] Security assessment of vendors
- [ ] Regular updates for dependencies
- [ ] Minimal permissions for integrations
- [ ] Monitoring for supply chain attacks
